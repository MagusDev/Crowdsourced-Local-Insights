Link Relations in GeoData API

This document defines the custom and common link relations used throughout the GeoData API. These relations help clients understand the semantics of available links and actions in the hypermedia responses.

GeoMeta Link Relations (Namespace: geometa)

Relation Description Methods
geometa:add-user Create a new user POST
geometa:add-insight Create a new insight POST
geometa:add-feedback Create feedback on an insight POST
geometa:insights-all List all insights (optionally filtered by location or category) GET
geometa:insights-by List insights created by a specific user GET
geometa:insight-feedbackcollection List all feedbacks related to an insight GET
geometa:user-feedbackcollection List all feedbacks created by the authenticated user GET
geometa:user-insightcollection List all insights created by a specific user GET
geometa:usercollection List all users (admin only) GET

Standard Link Relations

Relation Description
self Canonical link to the current resource
profile Link to the resource's profile document
edit Update the current resource
delete Delete the current resource
up Parent resource or collection
author Link to the creator/owner of the resource

Authentication Link Relations

Relation Description Methods
auth:login Submit API key or credentials for authentication POST
